## 一、项目定义

### 1.1 我们是什么

**LLM 驱动的 AI 漫剧全链路生产平台**

> 原定位（v4.35 前）：集成了 ComfyUI 的无限画布工具。
> 2026-07-20 修正：基于 WorkBuddy 调研（85 来源）+ 方向校准分析，重新定义为全链路漫剧平台。

```
用户输入（自然语言/小说/IP设定）
  ↓ LLM 意图解析（DeepSeek v4）
  ↓ 工作流蓝图组装（34+ 蓝图库 → ComfyUI JSON）
  ↓ ComfyUI 执行（WebSocket 实时反馈）
  ↓ Konva 三层画布展示（结果可视化 + 血缘图谱 + 可编辑）
```

### 1.2 核心架构

```
frontend/ (React + TypeScript + Konva)                    ✨ v5.1 端口可视化 + 右键菜单
    ├── Canvas.tsx        — Konva 画布（拖拽/缩放/端口连线/文本/音频渲染）
    ├── ControlPanel.tsx  — 14 种节点模式 + 参数面板 + 首尾帧上传
    ├── WorkflowGraph.tsx — 工作流 DAG 可视化
    ├── Storyboard.tsx    — 分镜时间轴
    ├── Timeline.tsx      — 视频拼接时间轴
    ├── store.ts          — Zustand 状态管理（含 portEdges 端口连线）
    ├── types.ts          — NodeKind(9种) + PortEdge + port 定义
    └── api.ts            — 21 个 API 调用封装
        ↓ HTTP + WebSocket
agent/ (Python FastAPI)
    ├── main.py           — API 路由（21 端点）
    ├── deepseek.py       — LLM 意图解析（JSON 严格模式）
    ├── intent_map.py     — 12 种 action → 34+ 工作流蓝图路由
    ├── comfy_client.py   — ComfyUI WebSocket 通信 + 进度回传
    ├── video_blueprints.py — Wan2.2 Bernini 视频蓝图（v5.0）
    ├── workflow_executor.py — ⏳ 端口连线拓扑排序引擎
    ├── agent_cli.py      — ⏳ `ic` 命令行 Agent 入口
    └── env_check.py      — 环境自诊断
        ↓ WebSocket
ComfyUI (本地推理引擎)
    ├── 2257 节点 + 9 自定义节点
    ├── SDXL/FLUX/Wan2.2 Bernini 全系列
    ├── CosyVoice2 / MusicGen (音频)
    └── IPAdapter 全家桶 → StoryDiffusion 升级中
```

## 三、v5.0 战略方向与竞品定位

### 3.1 三层画布模型（v5.0 核心架构）

```
┌─────────────────────────────────────────────────────────┐
│  第一层：项目画布（Project Canvas）— 资产管理           │
│  - 角色/场景/道具/风格节点（实体注册表）               │
│  - 每个节点有独立属性面板（JSON Schema）                │
│  - 一致性策略按场景自动选择（StoryDiffusion 17 选 1）  │
├─────────────────────────────────────────────────────────┤
│  第二层：故事板画布（Storyboard Canvas）— 分镜编排     │
│  - 时间线/序列排列分镜                                 │
│  - 分镜引用第一层资产                                   │
│  - 自动生成 Engineering Prompt（注入一致性参数）        │
│  - 批量生成所有分镜 → 视频时间轴                       │
├─────────────────────────────────────────────────────────┤
│  第三层：工作流画布（Workflow Canvas）— 可编辑工作流   │
│  - 可视化 + 可编辑的 ComfyUI 拓扑                      │
│  - 乐高式蓝图组装（34+ 蓝图库）                        │
│  - 内置 Workflow Validator（skill 化）                 │
│  - 提交执行 → WebSocket 实时反馈                       │
└─────────────────────────────────────────────────────────┘
```

### 3.2 工作流引擎升级：蓝图 → 乐高式组装

```
用户自然语言
  ↓
Phase 1: 意图解析  → DeepSeek v4（已有）
  产出：结构化需求（目标/风格/角色数/分镜数/一致性需求）
  ↓
Phase 2: 工作流规划  → 新模块 workflow_planner.py
  从 34+ 蓝图库匹配 → LLM 辅助决定节点连接拓扑
  产出：节点 DAG
  ↓
Phase 3: 蓝图组装  → 新模块 workflow_assembler.py
  参数填入 + 一致性策略自动选择（17 选 1）
  + 量化策略匹配（fp8/GGUF Q5）+ 多角色同框自动注入
  产出：完整 ComfyUI JSON
  ↓
Phase 4: 校验  → comfyui-workflow-validator（已有）
  节点存在性检查 + 必需输入校验 + 生成引导图
  产出：合格 JSON
```

### 3.4 直接竞品对标（2026-07-20 更新）

> **关键新增**：NodeTool 是 GitHub 上唯一同形态的开源竞品，但其 AGPL-3.0 许可对商业使用构成"投毒"风险——任何使用必须全量开源。我们 Apache 2.0 是真实壁垒。

#### 3.4.0 NodeTool（⚠️ AGPL-3.0 商业毒性）

| 维度 | 详情 | 结论 |
|------|------|------|
| 形态 | 开源 Web 应用 (ComfyUI 内核) | 与我们同赛道，1.8K⭐ |
| 许可证 | **AGPL-3.0** | ❌ 商业产品无法使用，需全量开源 |
| 画布 | React Flow（非自研） | 不如 Konva 无限画布的自由度 |

## 十一、端口系统设计（v5.1 已完成）

> **详见** `分析调研/2026.7.20_1420LibTV使用指南_工程师创建/v5.0-架构重构方案.md` 问题一
>

### 11.1 NodePort 接口

```typescript
// types.ts
export interface NodePort {
  id: string;                    // 用 name 生成，如 "image_out"
  name: string;                  // "image_out" | "prompt_out" | "video_in" 等
  label: string;                 // 显示标签："图片" | "提示词" | "视频"
  type: 'image' | 'video' | 'text' | 'audio' | 'prompt' | 'control';
  direction: 'input' | 'output'; // 输入端口在左侧，输出端口在右侧
}

export interface PortEdge {
  id: string;
  fromPortId: string;
  toPortId: string;
  fromNodeId: string;
  toNodeId: string;
}
```

### 11.2 每种节点类型默认端口矩阵

> `defaultPortsForKind(kind: NodeKind): NodePort[]` — `types.ts` 中定义

| NodeKind | 输入端口 | 输出端口 |
|----------|---------|---------|
| `image` (文生图) | `prompt_in` `control_in` | `image_out` |
| `image` (图生图) | `image_in` `prompt_in` | `image_out` |
| `video` | `image_in` `prompt_in` `audio_in` | `video_out` |
| `control` (ControlNet) | `image_in` | `control_out` |
| `lora` | — | `lora_out` |
| `text` | — | `text_out` |
| `audio` | `text_in` | `audio_out` |
| `storyboard_frame` | `image_in` `prompt_in` | `frame_out` |

### 11.3 端口连线交互规范

1. **渲染**：输入端口在节点左侧，输出端口在节点右侧，等距排列
2. **类型颜色**：image=蓝 `#5b9bd5`，video=品红 `#ff6b9d`，text=青 `#00d4aa`，audio=紫 `#a46bff`，prompt=琥珀 `#ffa726`，control=绿 `#4caf50`
3. **拖拽连线**：从输出端口拖拽到输入端口，实时显示临时连线（红色虚线）
4. **自动匹配**：松手时按"最近端口距离"匹配同目标节点的输入端口
5. **Hover 高亮**：鼠标悬停节点时，该节点所有端口连线变为金色 `#ffa726`
6. **持久化**：PortEdge 存储于 `store.ts → portEdges[]`，支持 localStorage 序列化 + undo/redo

### 11.4 后端接口

```
POST /api/port-edge         创建端口连线
DELETE /api/port-edge/{id}  删除端口连线
GET  /api/port-edges        获取所有连线
```

## 十四、WorkflowExecutor 引擎设计（v5.2 已完成）

> **详见** `分析调研/2026.7.20_1420LibTV使用指南_工程师创建/v5.0-架构重构方案.md` 问题四
>
> v5.1 仅实现了前端「▶ 执行」按钮（POST `/api/workflow/execute-chain`），真正的拓扑排序引擎在 v5.2 实现。

### 12.1 设计目标

将画布上的节点和端口连线自动翻译为可执行的 ComfyUI 工作流链。

### 12.2 核心流程

```
1. 收集画布状态
   nodes[] (含 kind/mode/参数) + portEdges[] (连线拓扑)
        ↓
2. 拓扑排序 (graph.py → topological_sort)
   从 type=image 的输入端口出发，BFS 遍历 portEdges
   检测环 → 报错退出
   排序结果：执行顺序列表 [node_id, ...]
        ↓
3. 蓝图组装 (workflow_assembler.py)
   每个节点按 kind/mode 调对应蓝图函数
   端口连线映射为 ComfyUI 节点间的数据连接
        ↓
4. ComfyUI JSON 编译 (pipeline_compiler.py)
   多蓝图合并为单个 ComfyUI prompt JSON
   处理节点 ID 去重、latent 尺寸对齐
        ↓
5. 提交执行
   POST ComfyUI /prompt → WebSocket 进度 → 结果回写画布
```

### 12.3 API 接口

```
POST /api/workflow/execute-chain
  Body: {
    "nodes": CanvasNode[],      // 画布上所有节点
    "port_edges": PortEdge[]    // 端口连线
  }
  Response: {
    "status": "ok",
    "prompt_id": "xxx",
    "results": [
      { "node_id": "abc", "output_type": "image", "filename": "out_001.png" }
    ]
  }
```

### 12.4 拓扑排序伪代码

```python
# graph.py → topological_sort(nodes, port_edges) → list[str]

def topological_sort(nodes: list, port_edges: list) -> list[str]:
    in_degree: dict[str, int] = {}
    graph: dict[str, list[str]] = {}
    
    for node in nodes:
        in_degree[node.id] = 0
        graph[node.id] = []
    
    for edge in port_edges:
        graph[edge.fromNodeId].append(edge.toNodeId)
        in_degree[edge.toNodeId] += 1
    
    queue = [nid for nid, deg in in_degree.items() if deg == 0]
    result = []
    while queue:
        nid = queue.pop(0)
        result.append(nid)
        for neighbor in graph[nid]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    
    if len(result) != len(nodes):
        raise ValueError("检测到环：端口连线形成了循环依赖")
    return result
```
