## v4.23 — ControlNet 节点化

## v4.41 画布性能优化 — 视口裁剪 + 懒加载 + 历史限制 (2026-07-20)

### 需求背景
P2：当前画布无虚拟化——`nodes.map()` 无条件渲染所有节点，所有图片同时加载 `/api/image/{filename}`，撤销栈无限增长。当节点数超过 30-50 时，帧率下降、内存膨胀。

### 技术方案

#### 1. 视口裁剪（AABB + Viewport Culling）
- **碰撞算法**: AABB（轴对齐包围盒），用 `node.x/y/width/height` 与视口矩形做相交检测
- **缓冲边距**: `VIEWPORT_MARGIN = 600px`（canvas 坐标），防止平移时节点突然出现
- **阈值跳过**: 节点数 ≤ 20 时返回 `null`，直接全量渲染，避免裁剪开销 > 收益
- **过滤位置**: 在 `<Layer>` 的 `nodes.map()` 中 `if (visibleNodeIds && !visibleNodeIds.has(n.id)) return null`

#### 2. 图片懒加载
- `NodeImage` 新增 `isVisible?: boolean` prop，默认 `true`（兼容 ≤20 节点模式）
- `useEffect` 加载 `imageUrl(...)` 前先检查 `isVisible`，视口外直接 `return`
- 已加载过的图片不会卸载（Konva Image 元素保持缓存）

#### 3. 撤销历史深度限制
- `store.ts` 新增 `MAX_HISTORY = 50`
- `snapshot()` 中 `past` 超过限制时取最后 49 个快照 + 新快照，避免内存线性增长

### 改动清单（3 files changed）

| 文件 | 变更 |
|------|------|
| `frontend/src/store.ts` | + `MAX_HISTORY = 50` 常量；`snapshot()` 增加深度限制逻辑 |
| `frontend/src/Canvas.tsx` | + `useMemo` 导入 + `getVisibleNodeIds()` AABB 裁剪函数 + `isVisible` prop 链 + 图片懒加载 + 版本注释 v4.41 |
| `frontend/src/App.tsx` | 版本号 → v4.41 |

### 性能特征
- **小画布**（≤20 节点）: 零开销，visibleNodeIds 返回 null，全量渲染
- **大画布**（20+ 节点）: O(n) AABB 相交检测，每个节点仅一次比较
- **懒加载**: 视口外节点不发起 `Image()` 网络请求，减少带宽和内存
- **历史深度**: 撤销栈 ≤ 50 个快照，防止无限增长
