# v5.3 Agent 工作流路由 Bug 分析 (2026-07-20)

## Bug 1: 分镜生成错误 — "太空站"变"小狐狸"

**触发**: 用户输入"帮我做5个太空站的分镜"
**现象**: NoobAI-XL 生成动漫小狐狸女孩，而非太空站
**根因链**:
1. DeepSeek LLM 正确解析 storyboard → 5 shots(含英文 prompt) ✅
2. `IntentResponse` 模型无 `shots` 字段 → LLM 分镜数据被丢弃 ❌
3. 前端 `elements=[]` → 回退用 `intent.subject="帮我做5个太空站的分镜"` ❌
4. NoobAI-XL 收到中文文本 → 不理解 → 生成默认动漫内容 ❌
**修复**: IntentResponse + shots字段 / ChatPanel 优先用 shots.prompt

## Bug 2: "需要5张" → 只生成 1 张

**触发**: "创建一个白发红瞳的少女角色需要5张"
**根因**: LLM 解析 batch_size/count=5 ✅ → ChatPanel 不传 → 默认 batch_size=1
**修复**: ChatPanel 读取 `params.count` 传递给 `batch_size`

## Bug 3: "把这张图变成动态视频" → 生成图片

**触发**: "把这张图变成动态视频"
**根因**: 
1. ChatPanel 不分 action 调用同一个 `generate()` ❌
2. 没传 `input_image`（img2vid 需要已选画布节点）
3. 后端降级为 txt2img → 生成静态图片 ❌
**修复**: 
- ChatPanel 读 `intent.action` → 路由不同参数
- needsInput → 从 store.getAllSelectedIds() 取画布节点
- 无选中→提示用户"请先选中图片节点"

## Bug 4: Agent Skill 不思考创作工作流

**现象**: 所有请求走同一个工作流，不做差异化
**根因**: `comfy_client.py` 只有 `build_txt2img()` 一个构建方法，其他 action 降级
**方案**: 移植 comfyui-workflow-skill 的 34 模板 → 变成模板驱动构建
