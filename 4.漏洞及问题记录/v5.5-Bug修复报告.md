## B1 · ChatPanel 发送后必现崩溃

**症状**: `Cannot read properties of undefined (reading 'template_id')`

**根因**: `ControlPanel.tsx` L300 — LLM Intent API 响应中 `template_id` 字段为 `null` / `undefined`，前端未做兜底。

**影响范围**: ChatPanel 自然语言对话功能完全不可用（100% 必现）。

**修复**: `ControlPanel.tsx` L300
```ts
// 修复前
templateId: res.template_id,

// 修复后
templateId: res.template_id || 'unknown',
```
加 `|| 'unknown'` 兜底，即使后端不返回 `template_id` 也不会崩溃。

**验证**: 前端 vitest 40/40 passed，无回归。

## B2 · 长链工作流执行后节点 ports 无法重连

**症状**: 执行工作流链后，端口连线无法重新建立，需刷新页面。

**根因分析**:
1. **前端 `App.tsx` 执行链路三重缺陷**:
   - 未计算 `root_node_id`（拓扑起点），错误地将所有节点发送给后端
   - 响应格式匹配错误：用 `data.status === 'ok'` 判断，但后端返回 `data.results`
   - 执行完成后未将生成结果（`filename`）回写节点状态，导致端口重连时找不到输出文件

2. **`Canvas.tsx` 端口连接判断错误**:
   ```ts
   // 错误：connectedTo 是数组，非 null/undefined 比较
   port.connectedTo != null && port.connectedTo !== undefined
   
   // 正确：检查数组长度
   port.connectedTo != null && port.connectedTo.length > 0
   ```

3. **`store.ts` 端口连线状态不同步**:
   `addPortEdge` / `removePortEdge` 只更新 `portEdges` 数组，未同步更新 `nodes[*].ports[*].connectedTo`——导致 UI 渲染与实际状态脱节。

4. **`main.py` 响应格式缺字段**:
   `/api/workflow/execute-chain` 返回缺少 `"status": "ok"` 字段。

**修复**:

| 文件 | 修改 |
|------|------|
| `App.tsx` L362-391 | ① 拓扑起点查找 ② `data.results` 匹配 ③ `store.updateNode(r.node_id, {filename: r.output_file})` |
| `Canvas.tsx` L359/389 | `connectedTo.length > 0` 替换错误的 null 判断 |
| `store.ts` | `addPortEdge`/`removePortEdge` 同步 `nodes[*].ports[*].connectedTo` |
| `main.py` | 补充 `"status": "ok"` 字段 |

**验证**: 全量测试 362/362 passed。

## B3 · `list_loras()` / `list_controlnets()` 返回空列表

**症状**: API 端点 `/api/loras` 和 `/api/controlnets` 始终返回空数组。

**根因**: `comfy_client.py` 被直接导入时（不通过 `main.py`），`.env` 文件未被加载，`SHARED_MODEL_LIB` 环境变量为空字符串，导致 `list_loras()` 路径 `""/loras` 不存在 → 返回 `[]`。

**影响范围**: 前端 LoRA 选择器、ControlNet 选择器为空；`list_checkpoints()` 同样受影响。

**修复**: 在 `comfy_client.py` 顶部添加 `_load_dotenv_comfy()` 函数，模块初始化时自动加载 `.env`。

**验证**:
```
修复前: Loras: 0, ControlNets: 0, Checkpoints: 0
修复后: Loras: 16, ControlNets: 2, Checkpoints: 3
```
