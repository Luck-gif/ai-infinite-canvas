## 二、Bug 修复清单（3 项全部修复）

### Bug #1 — 控制面板分镜输入框背景色错误（Critical）

| 项目 | 详情 |
|------|------|
| **位置** | `ControlPanel.tsx` 第 1127 行 |
| **问题** | `theme.surface.input` — `theme.surface` 属性不存在 |
| **根因** | `theme.ts` 中 bg.input 存在（#141f33），但 surface 对象从未定义 |
| **影响** | `theme.surface` 为 `undefined` → `theme.surface.input` 抛出 `TypeError` → 分镜 textarea 渲染无背景色 → 暗黑主题下显示为白色方块 |
| **修复** | `theme.surface.input` → `theme.bg.input` |

### Bug #2 — 分镜生成传参使用不存在的 GenParams 字段（Medium）

| 项目 | 详情 |
|------|------|
| **位置** | `ControlPanel.tsx` 第 461 行 |
| **问题** | `checkpoint: p.ckpt \|\| undefined` — `p.ckpt` 不在 GenParams 类型中 |
| **根因** | GenParams 的 checkpoint 字段名为 `model`（类型：`'' \| 'qwen2' \| 'sdxl'`），不是 `ckpt` |
| **影响** | `p.ckpt` 运行时恒为 `undefined` → 分镜请求的 checkpoint 参数始终为 `undefined` → 无法指定模型切换，始终使用默认模型 |
| **修复** | `p.ckpt \|\| undefined` → `p.model \|\| undefined` |

### Bug #3 — 未使用变量 `base`（Low）

| 项目 | 详情 |
|------|------|
| **位置** | `ControlPanel.tsx` 第 477 行 |
| **问题** | `const base = nodeCount;` 声明后从未使用 |
| **影响** | 分镜帧节点位置未考虑已有节点偏移（固定从 x:60 y:60 开始，大量节点时可能重叠） |
| **修复** | 删除未使用声明；分镜帧放置位置重叠需后续改进 |
